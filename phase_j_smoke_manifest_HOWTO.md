# Smoke-test workflow + backend disclosure

This drop adds two things to `~/Desktop/pcg-neurips2026/`:

1. **`make smoke-full`** — one command runs all 5 experiments with the
   deterministic mock backend, regenerates every figure, every table,
   and the disclosure manifest. ~3-5 min on M4 Pro, no GPU, no network.
2. **`scripts/build_backends_manifest.py`** — walks `results/` and emits
   `artifacts/backends_manifest.{json,md,tex}` so reviewers can see
   exactly which LLM was used for each experiment.

## What's in the tarball

```
Makefile                                  REPLACE  Adds smoke-full + manifest targets
scripts/build_backends_manifest.py        NEW      Auto-disclosure builder
HOWTO.md                                           This file
```

Nothing else changes. No new dependencies.

## Apply

```bash
cd ~/Desktop/pcg-neurips2026
tar xzf "$HOME/Downloads/phase j smoke and manifest.tar"   # or whatever the
                                                            # filename ends up
rsync -av --exclude='HOWTO.md' phase_j_smoke_manifest/ ./   # if rsync available
# OR
cp -r phase_j_smoke_manifest/Makefile ./
cp -r phase_j_smoke_manifest/scripts/build_backends_manifest.py ./scripts/
rm -rf phase_j_smoke_manifest
```

Verify the new targets are visible:

```bash
make help
```

You should see the new `smoke-full` and `manifest` lines.

## Run the full smoke test

```bash
source multi-agents/bin/activate
make smoke-full
```

This runs:
1. R1 audit decomposition (mock, 20 examples × 3 seeds)
2. R2 redundancy law (mock, 20 examples × 3 seeds)
3. R3 responsibility (mock, 20 examples × 3 seeds)
4. R4 cost-vs-harm (mock, 20 examples × 3 seeds)
5. R5 overhead (mock, 20 examples × 3 seeds)
6. Figures (intro_hero, summary_benchmark, R1-R5 individuals)
7. Tables (LaTeX)
8. Backend disclosure manifest

After it finishes, you'll have:

Every figure has the diagonal "MOCK BACKEND · PREVIEW ONLY" watermark
because all runs were on the mock backend. This is the protective
mechanism — reviewers see at a glance that the figures aren't real-LLM
results until you re-run on Colab/H100 and the watermark disappears.

## Adjust scope

To run more examples per seed:

```bash
make smoke-full SMOKE_N=100   # ~10-15 min, still mock
```

To run a single experiment with a real LLM (one-off):

```bash
make r1 BACKEND=hf_local SEEDS="0 1 2"
make manifest                  # rebuild disclosure to reflect the change
```

## Backend disclosure — what reviewers see

The manifest shows ONE row per experiment, listing the backend kind,
model, dataset, n_examples, seeds, and timestamp. Mixed-backend papers
(e.g. "we ran R1 on Qwen-7B but R5 on Llama-3.3-70B for cost reasons")
are first-class — the manifest just lists what you actually did.

Sample after a partial real run:

| Experiment | Backend | Model | Dataset | N | Seeds |
|---|---|---|---|---|---|
| **R1** | 🟢 HF local | `Qwen/Qwen2.5-7B-Instruct` | hotpotqa | 200 | 0..4 |
| **R2** | 🟢 HF local | `Qwen/Qwen2.5-7B-Instruct` | hotpotqa | 200 | 0..4 |
| **R3** | 🟢 HF local | `deepseek-ai/deepseek-llm-7b-chat` | hotpotqa | 200 | 0..2 |
| **R4** | 🟡 mock | `(none)` | synthetic | 50 | 0 |
| **R5** | 🟢 HF Inference | `meta-llama/Llama-3.3-70B-Instruct` | hotpotqa | 50 | 0..2 |

The R4 row being mock is fine — you'd note in the paper "R4 uses
synthetic data because the cost-harm tradeoff is dataset-independent".
But reviewers see the disclosure honestly.

## Wiring into the paper

Add to your appendix:

```latex
\section{Backend disclosure}
\input{artifacts/backends_manifest.tex}
```

Add to README.md:

```markdown
## Reproducibility

See [`artifacts/backends_manifest.md`](artifacts/backends_manifest.md) for
the LLM backend used in each experiment. Smoke-test the full pipeline
end-to-end (no GPU required) with `make smoke-full`.
```

Both files are auto-regenerated by `make manifest`, so they always
reflect the latest run. Add `artifacts/` to git (it's small, ~2 KB total)
so the disclosure is part of the source of truth.

## CI integration

Add this single line to your GitHub Actions workflow to keep the
disclosure honest in CI:

```yaml
- run: make smoke-full
- run: git diff --exit-code artifacts/   # fails if disclosure stale
```

## What this does NOT do

- It doesn't run real LLMs. That's deliberate — the smoke test must be
  free, fast, and local. For real-LLM runs, use the Colab+H100 workflow
  documented earlier.
- It doesn't generate the demo app's bundled traces (those are baked
  into `app/demo_data/`). Re-run the demo app's seed-data script if you
  want fresh demo traces.
- It doesn't push to git. Use the rsync-then-commit workflow you
  already have.
